## Cross-cutting concerns

### Authentication / session handling
SessionMiddleware (Redis-backed, `backend/src/app/core/security.py`) enforces HttpOnly/Secure/SameSite=Strict cookies and validates Origin headers on all state-changing requests. Session idle timeout is configurable (default 8h). F-002 fixed Redis connection caching + lifespan shutdown. Verified by T-010, T-031, T-032, T-213, and `test_session_middleware_redis_lifecycle.py`.

### Read-only execution invariant
Dual defense: (1) `SourceDBConnector` connects via a read-only PostgreSQL role, and (2) the evaluator's `ReadOnlyRule` rejects any SQL containing data-modifying keywords before execution. Verified by `test_source_db_connector.py`, `test_executor.py`, and `test_unsafe_pattern_blocked.py`.

### LLM provider lifecycle
F-001 fixed the per-request adapter leak: `LLMProviderFactory` now caches adapter instances, and the FastAPI lifespan closes all `httpx.AsyncClient`s on shutdown. F-008 added `LLM_MODEL_NAME` to `Settings`. Verified by `test_adapter_lifecycle.py` and `test_us5_operator_effort.py`.

### i18n key coverage
`en.json` is the single source of truth for English strings. F-004 tightened the E2E missing-key regex to catch 2-segment keys. F-006 extended the ESLint `no-inline-user-strings` rule to cover template literals and JSX expression containers. F-009 aligned `ar.json` key set with `en.json` (36 untranslated placeholders acknowledged — Arabic translation is deferred to Phase 4 per Constitution §11). Verified by T-179, T-182, T-183, T-184, T-185.

### RTL readiness
T-180 (Stylelint physical-direction audit), T-181 (Tailwind output scan), and T-186 (E2E RTL layout regression) confirm zero hardcoded directional CSS in user-facing components. All layout uses logical utilities (`ms-*`, `me-*`, `ps-*`, `pe-*`). F-005 added computed-style assertions to T-186. Verified clean.
